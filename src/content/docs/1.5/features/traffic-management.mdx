Nantian Gateway 1.5 uses Kubernetes Gateway API resources as the primary traffic-management interface.

## Route Types

| Traffic | Resource | Use case |
| --- | --- | --- |
| HTTP and HTTPS | `HTTPRoute` | REST APIs, web apps, matching, filters, and traffic splitting. |
| gRPC | `GRPCRoute` | Service and method routing for gRPC workloads. |
| TCP | `TCPRoute` | Databases, queues, and custom TCP protocols. |
| UDP | `UDPRoute` | DNS, telemetry, media, and custom UDP protocols. |
| TLS passthrough | `TLSRoute` | SNI-based forwarding when backends terminate TLS themselves. |

## Weighted Traffic Splitting

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: canary
spec:
  rules:
    - backendRefs:
        - name: api-v1
          port: 8080
          weight: 90
        - name: api-v2
          port: 8080
          weight: 10
```

## Backend Load-Balancing Policy

`BackendLBPolicy` is an experimental Gateway API policy for backend balancing.

Supported strategy values are `RoundRobin`, `ConsistentHash`, `LeastRequest`, and `Random`. For `ConsistentHash`, supported key types are `SourceIP`, `Header`, and `Hostname`.

```yaml
apiVersion: gateway.networking.k8s.io/v1alpha2
kind: BackendLBPolicy
metadata:
  name: echo-lb
spec:
  targetRefs:
    - group: ""
      kind: Service
      name: echo
  loadBalancing:
    type: ConsistentHash
    consistentHash:
      keyType: Header
      headerName: x-session-id
```
