Security and observability are part of the gateway path. Nantian Gateway 1.5 combines Gateway API security resources with runtime diagnostics from the control plane and data plane.

## TLS And Backend Validation

Nantian Gateway supports TLS termination on `Gateway` listeners, TLS passthrough with `TLSRoute`, backend TLS validation with `BackendTLSPolicy`, and mixed listener designs.

## Cross-Namespace Controls

Use `ReferenceGrant` when a route in one namespace must reference a backend, certificate, or other resource in another namespace. Keep grants narrow and review them during application onboarding and offboarding.

## Runtime Visibility

| Surface | Purpose |
| --- | --- |
| Health probes | Kubernetes liveness and readiness checks. |
| Admin API | Runtime config, connected clients, backend state, and diagnostics. |
| Prometheus metrics | Request, route, backend, xDS, TLS, AI, and Wasm signals where enabled. |
| Grafana dashboards | Visual entry points for gateway health and traffic behavior. |
| Alerting rules | Availability, latency, error, and certificate risk signals. |
| Troubleshooting docs | Known failure modes and inspection commands. |

AI Gateway and Wasm plugins add feature-specific signals such as token usage, policy decisions, plugin load failures, plugin rejections, and hook latency. These capabilities remain experimental where they depend on `AIService`, `TokenPolicy`, or `WasmPlugin`.
