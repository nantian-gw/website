---
title: Helm 安装
---

Helm chart 是安装 Nantian Gateway 的推荐方式。它把控制面、数据面、dashboard、RBAC、NetworkPolicy、Service、ConfigMap 和默认 `GatewayClass` 打包到同一个 release。

## 前提条件

先安装 Gateway API standard CRD：

```bash
kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.5.1/standard-install.yaml
kubectl get crd gateways.gateway.networking.k8s.io
```

你还需要 Helm 3.x，以及能访问目标集群的 `kubectl`。

## 添加 Chart 仓库

使用当前 chart 仓库地址：

```bash
helm repo add nantian-gw https://chart.nantian.dev
helm repo update
```

不要使用旧的复数形式 chart host。

默认搜索只显示稳定 chart 版本。每个 commit 生成的 snapshot chart 会以 `0.2.3-<git-sha>` 这类 prerelease 版本发布；需要查看或安装这些版本时加上 `--devel`：

```bash
helm search repo nantian-gw/nantian-gw --versions --devel
```

## 使用默认值安装

```bash
helm install nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw \
  --create-namespace
```

默认 release 会创建命名空间 `nantian-gw`、2 个控制面副本、2 个数据面副本、1 个 dashboard 副本、安装概述中列出的固定 Service，以及名为 `nantian-gw` 的 `GatewayClass`。

安装前可以先预览渲染结果：

```bash
helm template nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw > rendered.yaml
helm install nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw \
  --create-namespace \
  --dry-run
```

## 关键默认值

以下片段与当前 chart 默认值一致。

### 功能模式与 Gateway API CRD

```yaml
featureMode: standard
gatewayAPI:
  installCRDs: false
  channel: standard
```

`featureMode` 控制 Nantian Gateway 自身的运行时功能开关。`gatewayAPI.installCRDs` 控制本 Chart 是否渲染官方 Gateway API CRD。生产默认值保持 standard 运行时模式，并要求安装网关前已由平台层准备 Gateway API CRD。

### 全局与 GatewayClass

```yaml
global:
  imageRegistry: "ghcr.io"
  imagePullSecrets: []
  commonLabels: {}

gatewayClass:
  enabled: true
  controllerName: "gateway.networking.k8s.io/nantian-gw"
```

Chart 会根据 controller name 推导默认 `GatewayClass` 名称，因此 `gateway.networking.k8s.io/nantian-gw` 会变成 `nantian-gw`。

### 控制面

```yaml
controlplane:
  enabled: true
  replicas: 2
  image:
    repository: "nantian-gw/nantian-controlplane"
    tag: "latest"
    pullPolicy: Always
  config:
    grpcAddr: ":18080"
    adminAddr: ":18081"
    metricsAddr: ":18082"
    healthProbeAddr: ":18083"
    features:
      enableExperimentalGateway: false
      enableAiGateway: false
```

当前 chart 默认对控制面也使用 `latest` tag，方便跟随当前公开镜像快速安装。生产环境请在 values 文件里显式改成不可变 tag 或 digest。

控制面 Service 拆分如下：

| Service | 端口 |
| --- | --- |
| `nantian-gw-controlplane-grpc` | `18080` |
| `nantian-gw-controlplane-admin` | `18081` |
| `nantian-gw-controlplane-metrics` | `18082` |

### 数据面

```yaml
dataplane:
  enabled: true
  replicas: 2
  image:
    repository: "nantian-gw/dataplane"
    tag: "latest"
    pullPolicy: Always
  resources:
    requests:
      cpu: "250m"
      memory: "256Mi"
    limits:
      cpu: "2000m"
      memory: "1Gi"
  config:
    adminAddr: "0.0.0.0:19080"
    runtime:
      httpListenAddr: "0.0.0.0:10080"
  accessLogVolume:
    enabled: true
    name: access-logs
    mountPath: /var/log/nantian-gw
    sizeLimit: 256Mi
```

数据面 Service 是 `nantian-gw-dataplane-admin` 和 `nantian-gw-dataplane-metrics`，两者 Service port 都是 `19080`。运行时 HTTP 流量由配置中的监听地址 `0.0.0.0:10080` 处理。

### Dashboard

```yaml
dashboard:
  enabled: true
  replicas: 1
  image:
    repository: "nantian-gw/dashboard"
    tag: "latest"
    pullPolicy: Always
```

当前 chart 默认对 controlplane、dataplane 和 dashboard 都使用 `latest` tag。这个默认值适合快速验证安装链路，不适合作为生产环境的供应链控制策略；生产环境应在 values 文件里显式固定不可变 tag 或 digest。

不需要 Web UI 时可以关闭 dashboard：

```bash
helm upgrade --install nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw \
  --create-namespace \
  --set dashboard.enabled=false
```

### HPA、ServiceMonitor 与 NetworkPolicy

```yaml
hpa:
  enabled: false

serviceMonitor:
  enabled: false

networkPolicies:
  enabled: true
```

只有在集群已安装 Prometheus Operator CRD 时，才开启 `serviceMonitor.enabled`。HPA 默认关闭；启用前请确认集群有 metrics-server，并且已经验证过数据面的资源 requests。

## 常见覆盖

需要保留的修改请写入 values 文件：

```bash
helm upgrade --install nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw \
  --create-namespace \
  -f my-values.yaml
```

私有镜像仓库和副本数覆盖示例：

```yaml
global:
  imageRegistry: "registry.example.com"
  imagePullSecrets:
    - name: registry-credentials

dataplane:
  replicas: 4
```

## 实验功能

实验性 Gateway runtime 行为由 `featureMode` 控制。AI Gateway 仍是显式的控制面功能，并且需要 experimental 模式：

```yaml
featureMode: experimental
gatewayAPI:
  installCRDs: true
  channel: experimental
controlplane:
  config:
    features:
      enableAiGateway: true
```

`gatewayAPI.installCRDs` 和 `gatewayAPI.channel` 只控制 CRD 渲染。当需要由 Chart 渲染官方 experimental Gateway API CRD bundle 时，使用 `gatewayAPI.channel: experimental`。

完整的启动、CRD 和验证流程见[实验功能](/zh/configuration/experimental-features/)。

## 验证 Release

```bash
kubectl get pods -n nantian-gw
kubectl get gatewayclass nantian-gw
kubectl get svc -n nantian-gw
helm status nantian-gw -n nantian-gw
```

如果数据面没有就绪，请检查它是否能连接 `nantian-gw-controlplane-grpc:18080`，并查看控制面和数据面日志。

## 升级与卸载

升级已有 release：

```bash
helm upgrade nantian-gw nantian-gw/nantian-gw \
  --namespace nantian-gw \
  -f my-values.yaml
```

删除 release：

```bash
helm uninstall nantian-gw -n nantian-gw
kubectl delete namespace nantian-gw
```

Helm uninstall 会删除 chart 管理的资源。Gateway API CRD 和你在 chart 外创建的应用资源不会被 chart uninstall 删除。
