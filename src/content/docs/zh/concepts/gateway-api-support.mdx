title: Gateway API 支持矩阵
---

Nantian Gateway 目标对齐 Kubernetes Gateway API v1.5.1 规范。本页汇总当前对所有上游功能的支持情况，包括实验性功能与扩展功能。

## 阅读指南

- **默认支持** — 当 `enableExperimentalGateway=false` 时声明的功能。
- **实验性运行时支持** — 当 `enableExperimentalGateway=true` 时声明的功能。
- **当前状态** — Nantian Gateway 在运行时按条件启用的支持状态，数据来源为 `internal/gwapi/supported_features.go` 和一致性工作流。

:::note
核心功能（`Gateway`、`HTTPRoute`、`GRPCRoute`、`Mesh`、`ReferenceGrant`、`BackendTLSPolicy`、`TLSRoute`）已完整支持，此处不再重复列出。
:::

## Nantian Gateway 自有声明

除上游 Gateway API 功能集外，Nantian Gateway 额外声明：

| 功能 | 描述 |
|------|------|
| `TCPRoute` | TCP 路由支持 |
| `BackendLBSessionPersistence` | 后端负载均衡的会话保持 |
| `UDPRoute` | UDP 路由支持 |

## 启用实验性功能

要启用实验性运行时功能，设置以下 Helm 值：

```yaml
gatewayAPI:
  enableExperimentalGateway: true
```

进行含实验性功能的一致性测试：

```bash
CONFORMANCE_EXPERIMENTAL=true ALL_FEATURES=true make conformance
```
